The architectural truth this session uncovered, restated cleanly: **a Next.js App Router page is dynamic if ANY component in its render tree calls a dynamic function**. The store had at least four independent cookie reads (data layer via createClient, auth via supabase.auth.getUser in SiteHeader, i18n via getRequestConfig, and additional reads in the root layout). Each one alone is sufficient to defeat ISR. The fix required eliminating ALL of them. We discovered them sequentially, one fix revealing the next, until the last cookie was the load-bearing piece. Future "make this page static" work should start with `grep -rn "cookies()" app components lib` to inventory all blockers up front.
